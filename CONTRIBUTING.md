# Contributing to eyeprep

Thank you for your interest in contributing to eyeprep! We're excited to collaborate with you. All contributions, no matter how small, are valued and appreciated.

## Types of Contributions We Welcome

We welcome contributions of all kinds, including:

- **Bug reports**: Found something that doesn't work? Let us know!
- **Feature requests**: Have an idea for improving eyeprep?
- **Code improvements**: Submit fixes, optimizations, or new features
- **Documentation**: Improve our docs, add examples, or clarify existing content
- **Tests**: Help us improve test coverage
- **Feedback**: Share your experience using eyeprep

## How to Contribute

### Reporting Issues or Requesting Features

1. Check [existing issues](../../issues) to see if your issue or feature request has already been reported
2. If it hasn't, [open a new issue](../../issues/new) with:
   - A clear, descriptive title
   - A detailed description of the problem or feature
   - Steps to reproduce (for bugs)
   - Your environment details (Python version, operating system, eyeprep version)

### Submitting Code Changes

We follow a standard fork and pull request workflow:

1. **Fork the repository** on GitHub
2. **Clone your fork** to your local machine:
   ```bash
   git clone https://github.com/your-username/eyeprep.git
   cd eyeprep
   ```
3. **Create a new branch** for your changes:
   ```bash
   git checkout -b descriptive-branch-name
   ```
4. **Set up your development environment** (see the [Development Guide](DEVELOPMENT.md))
5. **Make your changes** and commit them with clear messages:
   ```bash
   git commit -m "Brief description of changes"
   ```
6. **Push to your fork**:
   ```bash
   git push origin descriptive-branch-name
   ```
7. **Submit a pull request** to the main repository with a description of your changes

### Before Submitting a Pull Request

- Make sure your code follows the project's style guidelines (we use linting and formatting tools)
- Add or update tests for your changes
- Update documentation if necessary
- Ensure all tests pass locally
- Check that your branch is up to date with the main branch

## Development Guidelines

For detailed instructions on setting up your development environment, code style expectations, and our testing practices, please see the [Development Guide](DEVELOPMENT.md).

## Questions or Need Help?

- Check our [documentation](https://eyeprep.readthedocs.io/) first
- Open a [GitHub Discussion](../../discussions) if you have questions about using eyeprep
- Feel free to ask for clarification on issues or in pull requests
